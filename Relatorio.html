Descrição geral do problema

O problema inicial é que é difícil para uma pessoa chegando numa nova cidade de saber onde ir para pedir carona: a probabilidade de ter um carro parando depende de vários factores que são difíceis de adivinhar (velocidade dos carros, lugar para parar, visibilidade do lugar, ...).

Assim, a ideia é de criar um aplicativo para celular indicando os melhores lugares para pedir carona.

O aplicativo é colaborativo: são os usuários que adicionam as informações que vão usar os outros. O principio é de usar um mapa Google com pontos compartilhados.

Tem 3 tipos de uso:

- indicar um novo ponto no mapa onde é fácil conseguir uma carona

- recomendar ou não um ponto que já existe

- olhar o mapa no lugar escolhido para decidir onde pedir uma carona

Cada ponto tem um valor associado (1 na criação, depois mais 1 com cada recomendação, e menos 1 no caso contrário), assim o usuário pesquisando um lugar tem uma ideia da confiabilidade de cada ponto.

Análise crítica

Funcionalidades

As funcionalidades implementadas funcionam bem, e são suficientes para usar o sistema numa situação real.

Contudo, umas outras funcionalidades poderiam ser implementadas para ter um melhor aplicativo, como:

- poder adicionar comentários a um ponto

- especificar para ir em qual direcção é o ponto

- poder escolher a confiabilidade dos usuários

- dar uma importância maior ao voto dum usuário confiável

- fazer estatísticas sobre as áreas mais eficientes

- decrementar a confiabilidade dum ponto se a ultima recomendação é antiga demais

Desempenho

O desempenho parece bom, mas não fizemos de testes com um grande número de clientes.

Uma crítica que podemos fazer é que o servidor não identifique individualmente cada cliente, assim ele não sabe quem mandou a informação dum novo ponto, e notifica ele de volta, ainda que não serve para nada.
